# BaseScene 原型搭建规范

## 1. 目标

本规范用于统一 `BaseScene` / 基地 Hub 的 **blockout / greybox / ProBuilder 原型搭建方式**，避免以下问题：

- 尺度不统一，人物移动和交互距离失真
- 命名混乱，后续替换正式资产时难以定位
- 场景结构不清晰，商人 / 任务 / 仓库 / 准备区互相污染
- 原型阶段做了过多“假美术”，导致返工成本高

本规范服务于：

- 阶段 `2.1 基地场景设计与搭建`
- 阶段 `2.2 商人 NPC 交互系统`
- 后续基地正式化、任务 NPC、剧情点位、设施升级

---

## 2. 原则

### 2.1 先功能，后美术

基地原型阶段只验证：

- 功能分区是否清晰
- 玩家动线是否顺畅
- 交互点是否容易找到
- 后续系统是否有稳定挂点

暂不追求：

- 高精度装饰
- 复杂小道具堆砌
- 正式光照与氛围
- 精修材质

### 2.2 一切以“可替换”为前提

Blockout 物体必须允许后续被：

- ProBuilder 细化
- 正式 Mesh / Prefab 替换
- 关卡逻辑挂点复用

因此：

- 外壳、分区、交互、路标、灯光必须分组
- 不把功能挂点直接绑死在装饰物体上

### 2.3 颜色只表达功能，不表达美术完成度

颜色用于快速识别区域：

- 蓝：准备区
- 绿：仓库区
- 橙：商人区
- 紫：任务区
- 浅蓝：恢复区
- 灰：中性结构

---

## 3. 尺度规范

## 3.1 单位

- `1 Unity Unit = 1 米`

## 3.2 网格吸附

推荐 ProBuilder / Unity Snap：

- 大结构：`1.0m`
- 模块搭建：`0.5m`
- 微调：`0.25m`

不建议长期使用小于 `0.1m` 的吸附做 blockout。

## 3.3 空间尺寸基线

- 层高：`4.6m`
- 门楣高度：`3.2m`
- 主通道宽度：`4.0m`
- 次通道宽度：`3.0m`
- 主门洞宽度：`3.6m`
- 次门洞宽度：`2.4m`
- 商人柜台高度：`0.95m`
- 主要标识牌中心高度：`2.2m`

## 3.4 玩家体验底线

- 不能让玩家在基地里频繁卡角
- 主功能区之间最好能通过中央枢纽直接到达
- 任一关键功能区从出生点出发应当在数秒内可达

---

## 4. 场景层级规范

`BaseScene` 推荐层级：

```text
BH_BaseHubRoot
├─ 00_Shell
├─ 10_ReadyRoom
├─ 20_Warehouse
├─ 30_MerchantWing
├─ 40_MissionWing
├─ 50_RecoveryBay
├─ 60_Gameplay
├─ 70_Wayfinding
├─ 80_Lighting
└─ 90_Debug
```

说明：

- `00_Shell`：地面、天花、外墙、主承重体
- `10~50`：功能分区 blockout
- `60_Gameplay`：交互点、终端、商人锚点、出生点、任务点
- `70_Wayfinding`：导视牌、区域名、地贴
- `80_Lighting`：原型灯光
- `90_Debug`：测试标记、可视化辅助物

## 5. 命名规范

## 5.1 前缀

- `BH_`：基地通用物体
- `ZONE_`：区域标记
- `SPAWN_`：出生/返回点
- `INT_`：交互物
- `NPC_`：NPC 或商人预留点
- `SIGN_`：导视
- `LGT_`：灯光

## 5.2 示例

- `BH_Shell_Floor`
- `BH_ReadyRoom_Bench_A`
- `ZONE_Warehouse`
- `SPAWN_Departure`
- `INT_MerchantTerminal`
- `NPC_WeaponDealer_Anchor`
- `SIGN_MissionBoard`
- `LGT_Atrium_Main`

## 6. 分区规范

## 6.1 准备区

职责：

- 出击
- 配装
- 地图确认

必须具备：

- 出击终端
- 配装/简报桌
- 入口处的明显识别色

## 6.2 仓库区

职责：

- 仓储
- 武器柜
- 保护栏位相关操作

必须具备：

- 大件储物体块
- 明确的仓储感体块
- 可放置仓库终端的稳定平面

## 6.3 商人区

职责：

- 固定商人交互

必须具备：

- 至少 3~4 个商人柜台占位
- 每个柜台独立锚点
- 中间留足玩家停留空间

## 6.4 任务区

职责：

- 任务公告板
- 汇报点
- 后续任务 NPC / 剧情触发

必须具备：

- 中心公告板
- 左右或前后可扩展的 NPC 点位

## 6.5 恢复区

职责：

- 死亡返回点
- 轻量恢复叙事

必须具备：

- 与主循环衔接的出生点
- 明确区别于准备区的视觉提示

---

## 7. ProBuilder 工作流规范

## 7.1 推荐流程

1. 先搭 **外壳**
2. 再切 **主通道**
3. 再切 **四个主分区**
4. 再放 **交互锚点**
5. 最后补 **低成本识别体块**

## 7.2 允许使用的原型手段

- Cube / Plane / ProBuilder Shape
- 简单倒角、挤出、门洞
- 统一色块材质
- 低成本文字牌 / 指示牌

## 7.3 暂不建议

- 复杂布线和细碎装饰
- 高密度重复小件
- 为原型临时做大量 UV / 贴图
- 把交互逻辑写死在临时装饰上

---

## 8. 基地一期布局建议

采用 **中央枢纽 + 四向功能翼**：

- 北：准备区
- 东：仓库区
- 西：商人区
- 南：任务区
- 东南或东北：恢复区

优点：

- 认路成本低
- UI/交互入口分布稳定
- 后续剧情与 NPC 挂点清晰

---

## 9. 验收清单

在提交 BaseScene blockout 前，至少确认：

- [ ] 玩家可无卡顿穿过所有主通道
- [ ] 准备区 / 仓库区 / 商人区 / 任务区识别清晰
- [ ] 关键交互点位已预留
- [ ] 命名与层级符合规范
- [ ] 原型物体可被后续正式资产替换
- [ ] 场景中不存在大量无意义小碎块
- [ ] 颜色编码与区域职责一致
